# Anti AI Slop Review

## Workflow

1. For product UI design, critique, redesign, or optimization, use Lazyweb first unless the task is strictly prose editing or code review.
2. Compare the work against the project identity: practical, modular, readable, friendly, expert but not academic.
3. Check that this is the Wynde UX Research Guide, not the banking Pathway project.
4. Inspect the rendered UI or relevant code/content. Use Figma context when available.
5. Look for generic patterns: vague hero copy, decorative-only visuals, repetitive cards, weak hierarchy, one-note color, cramped mobile layout, inaccessible controls, or placeholder-feeling examples.
6. Propose concrete fixes tied to files or UI regions.

## Review Criteria

- Reading quality for long-form guide content.
- Docs navigation clarity: sidebar, ToC, active states, mobile menu.
- Component usefulness: callouts, checklists, templates, examples, related links.
- Visual restraint and polish: spacing, type, color, assets, focus states.
- Product relevance: clear connection to UX research and Wynde workflows.
